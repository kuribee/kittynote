## docker存储

1. 默认情况下，在运行中的容器里创建的文件，被保存在一个可写的容器层。如果容器被删除了，则数据也没有了。这个可写的容器层是和特定的容器绑定的，也就是这些数据无法方便的和其它容器共享
2. Docker主要提供了两种方式做数据的持久化
   - Data Volume, 由Docker管理，(/var/lib/docker/volumes/ Linux), 持久化数据的最好方式
   - Bind Mount，由用户指定存储的数据具体mount在系统什么位置
3. dockerfile中Volume命令可以指定要持久化的目录文件
4. docker run -v也可以持久化，
   - 不指定：容器指定目录。这一种和dockerfile中指定很像。匿名卷
   - 一个名字：容器指定目录。命名卷
   - 宿主机目录：容器指定目录。bind mount
5. 匿名卷和命名卷都存在 /var/lib/docker/volumes
6. 发生在构建镜像阶段，写入镜像元数据，表示：这个目录是“数据持久化目录”。运行容器时，如果没有 -v，Docker 会自动为这个目录创建一个匿名卷，容器删除后数据还在（但不好找）。它不能指定宿主机路径，只能写容器内路径。声明式，不是强制式，它只是“建议这个目录要持久化”，运行时依然可以覆盖。
7. 发生在容器启动时，是“用户在运行容器时”的行为。不影响镜像本身。可以覆盖镜像中的 VOLUME 声明，就算 Dockerfile 里写了 VOLUME，你在 run 时指定 `-v` 也会替代它。所以一般不在dockerfile中用volume。
8. VOLUME无法指定目录，这是因为主机目录通常情况下是相对主机的一个目录，意味着这个目录在不同主机间会变化，并且可能导致构建失败。如果通过Dockerfile指定，那么每次部署时都需要指定主机目录。
9. bind mount很方便的一点是可以在镜像中启动某种编译环境，将文件夹映射到容器后就可以方便编译，即使宿主机没有该编译环境
10. Docker的volume支持多种driver。默认创建的volume driver都是local。例如sshfs的driver，让docker使用不在同一台机器上的文件系统做volume
11. ● 如果指定了已经存在的卷，Docker会使用该卷。● 如果指定的卷不存在，Docker会创建一个卷。
12. 只要卷被容器引用，就不能删除。
